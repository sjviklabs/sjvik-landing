# stevenjvik.tech (brand-site) — Recruiter-Conversion Audit

**Date:** 2026-04-28
**Scope:** Astro 6 + React + Tailwind 4 + nanostores app deployed to Hostinger via FTP-Deploy-Action
**Repo:** [sjviklabs/brand-site](https://github.com/sjviklabs/brand-site)
**Framing:** Recruiter-first conversion, brand-aligned where additive

---

## TL;DR

The site is well-engineered: variant system (terminal / editorial), accent switching, theme + motion control, pre-hydration FOUC prevention, focus-visible rings, skip-link, ⌘K palette. Visually it's strong.

**The conversion gaps are content-side, not visual:**
- Hero has no email CTA → recruiter has to scroll/dig for contact
- Hero copy has zero quantified metrics → 8+ years is not a result, 40% MTTR / 3,000+ endpoints / 0 CJIS violations are
- Contact form is front-end only — submissions are silently dropped
- Primary tech stack (Splunk / SolarWinds / Python / CrowdStrike) is buried in `/about` instead of surfaced where recruiters scan
- Case studies linked only from work-section footer — should be a top-level nav item

The implementation pass on these is small and high-leverage. I've built three new direction variants (Console, Dossier, Field Notes) at `/variants/` that demonstrate options for re-shaping the hero around these gaps.

**Severity legend:** P0 = blocks recruiter conversion or fails a11y · P1 = visible weakness · P2 = polish

## 1. Recruiter conversion

### 1.1 [P0] Hero has no direct email/contact CTA
- **Where:** [src/components/sections/HeroTerminal.astro:59-71](../src/components/sections/HeroTerminal.astro#L59-L71), [src/components/sections/HeroEditorial.astro](../src/components/sections/HeroEditorial.astro)
- **Current:** 3 CTAs — résumé.pdf · see the work · ⌘K commands. None of them open mail or surface the email address.
- **Why it matters:** A recruiter who decided to email you in 5 seconds has to scroll past Work / Projects / Guides to reach Contact, OR open the résumé PDF and find the email there. That's a friction tax on the highest-value conversion event.
- **Fix:** Add a 4th CTA in both hero variants:
  ```astro
  <a class="cta" href={`mailto:${CONTACT.email}`}>
    <span aria-hidden>✉</span> {CONTACT.email}
  </a>
  ```
  Place between résumé and "see the work" so it's the second most-prominent action.

### 1.2 [P0] Contact form silently drops submissions
- **Where:** [src/components/sections/ContactSection.astro:42-68](../src/components/sections/ContactSection.astro#L42-L68), comment at line 119 says *"wire to real backend in Step 11"*
- **Current:** Form shows `[200 OK]` locally on submit but never sends. Recruiter who fills the form thinks they reached you.
- **Why it matters:** Lost leads. Worse than no form — a broken form is a dishonest signal.
- **Fix:** Three viable options, ordered by ease:
  1. **Replace with mailto** — `<a href="mailto:steven@stevenjvik.tech?subject=Hiring%20inquiry">` instead of form. Zero infra. Works everywhere.
  2. **Formspree** — `action="https://formspree.io/f/<id>"`, set up in 5 min, 50 free submissions/mo
  3. **Hostinger SMTP relay** — if you want to keep it native; needs server-side endpoint
  Recommend #1 for now, swap to #2 if volume justifies.

### 1.3 [P0] Hero has zero quantified metrics
- **Where:** [src/components/sections/HeroTerminal.astro:39-56](../src/components/sections/HeroTerminal.astro#L39-L56)
- **Current copy:**
  > "I secure the unglamorous parts of the stack."
  > "8+ years across enterprise NOC monitoring, incident response, and compliance. I build the runbooks, harden the systems, and keep the alerts honest..."
- **Why it matters:** "8+ years" is a duration, not a result. The actual numbers are in `EXPERIENCE` records (40% MTTR reduction, 3,000+ endpoints, 0 CJIS violations, 25% reduction in unauthorized access, 35% stronger posture) — they should be in the hero, where 70% of scanners stop.
- **Fix:** Add a metric ribbon directly under the headline:
  ```astro
  <ul class="metric-ribbon" aria-label="Career outcomes">
    <li><strong>40%</strong>MTTR reduction</li>
    <li><strong>3,000+</strong>endpoints monitored</li>
    <li><strong>0</strong>CJIS audit violations</li>
  </ul>
  ```
  Style with monospace numerals + accent color. Three units max — more dilutes.

### 1.4 [P1] Primary tech stack not surfaced on home
- **Where:** Hero in both variants; full skill list lives at [src/pages/about.astro](../src/pages/about.astro) via `SKILLS` from `content.ts`
- **Current:** Hero mentions "runbooks, alerts, harden" but never names the tools. Recruiter searching for "Splunk experience" doesn't see Splunk above the fold.
- **Why it matters:** SIEM/SOC roles screen by tool. The tools are ranked content for recruiters.
- **Fix:** Add a primary-stack pill row in the hero (same content as `NowCard` status rows but visually elevated):
  ```
  Splunk · SolarWinds · Python · CrowdStrike · NIST 800-53 · CJIS · Linux
  ```
  Use the existing `.tag` or `.pill` utility classes. 6-8 chips max.

### 1.5 [P1] Case studies only reachable from work-section footer
- **Where:** [src/components/sections/WorkSection.astro:36-40](../src/components/sections/WorkSection.astro#L36-L40)
- **Current:** Case study links sit inside individual experience tiles. The "Case studies" concept isn't a top-level destination.
- **Why it matters:** [/experience/sound-transit](../src/pages/experience/sound-transit.astro), [/experience/jblm](../src/pages/experience/jblm.astro), [/experience/nexus-lab](../src/pages/experience/nexus-lab.astro) are your strongest proof artifacts — narrative depth that a tile can't carry. They deserve top-billing.
- **Fix:** Two parts:
  1. Add `Case studies` to `Nav.astro` between `Work` and `Projects`, anchoring `#case-studies`
  2. New `CaseStudiesSection.astro` between `WorkSection` and `ProjectsSection` — 3-card row pulling from EXPERIENCE filtered by `caseStudyUrl != null`

## 5. What the site does well — keep

1. **Variant system architecture** — CSS-driven `data-variant` toggle with both heroes pre-rendered in DOM is elegant. SEO-crawlable, no React flash, sharable URLs work.
2. **Pre-hydration script** — reading localStorage and writing `data-*` attributes before first paint prevents FOUC. Worth preserving on any new variant.
3. **Token system** — `@theme` block with full dark/light parity + accent overrides + bare-name aliases is well-organized. New components should reuse `--green`, `--ink`, `--fg-mute` etc., not introduce new colors.
4. **Skip link + focus rings** — already there. WCAG-respectful.
5. **Content single-source-of-truth** — [src/data/content.ts](../src/data/content.ts) is the right pattern. Any new sections should consume from here, not re-enter copy.
6. **Command palette** — ⌘K is a power-user signal that suits the audience.

## Three new direction variants

See [variants/index.html](./variants/index.html) for a side-by-side comparison. Each addresses every P0 above and proposes a different visual direction *distinct from the existing terminal and editorial variants*:

- **[Console](./variants/console/index.html)** — multi-pane NOC HUD dashboard. Spatial layout (sidebar + main + tickers), live-feeling status panels. Where terminal is text-on-screen, console is a control room. Strong fit for SOC/SRE roles.
- **[Dossier](./variants/dossier/index.html)** — federal candidate-file aesthetic. Stamped section labels (§ I / II / III), metadata table, document grid. Distinctive for federal / regulated-industry roles where the content surface itself signals fit.
- **[Field Notes](./variants/fieldnotes/index.html)** — engineering journal / handwritten-marginalia feel. Datestamps, "what I learned" voice, sketch-style annotations. Humanizing — communicates a thinking practitioner, not a stat sheet. Contrarian to the entire "AI portfolio" aesthetic. **✓ Shipped 2026-05-01.**

When you pick a direction, the implementation pass is straightforward:
- New `Hero{Console,Dossier,FieldNotes}.astro` component
- Add `"console"` (or chosen) to `Variant` type in [src/components/react/store.ts](../src/components/react/store.ts)
- Add CSS toggle rule in [src/pages/index.astro](../src/pages/index.astro) display table
- Add command in [CommandPalette.tsx](../src/components/react/CommandPalette.tsx) and chip in [TweaksPanel.tsx](../src/components/react/TweaksPanel.tsx)
